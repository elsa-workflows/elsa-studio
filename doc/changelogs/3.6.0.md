# Elsa Studio 3.6.0

Compare: [`3.5.3...3.6.0`](https://github.com/elsa-workflows/elsa-studio/compare/3.5.3...3.6.0)

---

## ⚠️ Breaking Changes / Upgrade Notes

- **`Elsa.Studio.WorkflowContexts` module removed**: The `WorkflowContexts` module has been moved to [elsa-extensions](https://github.com/elsa-workflows/elsa-extensions). ([c08392dec1](https://github.com/elsa-workflows/elsa-studio/commit/c08392dec1))

- **`Elsa.Studio.Labels` module reconstituted**: Labels was removed and re-introduced as a standalone module without an external package reference (previously bundled differently). If you were referencing a dedicated Labels NuGet package, update your setup to use the new module registration. ([ad346fc536](https://github.com/elsa-workflows/elsa-studio/commit/ad346fc536)) ([#712](https://github.com/elsa-workflows/elsa-studio/pull/712))

- **`IBrandingProvider` interface extended**: New properties (`AppName`, `AppVersion`, `Tagline`, `LoginBackgrounds`, `DefaultAppBarIcons`) were added to `IBrandingProvider`. If you have a custom implementation, update it to include the new members or derive from `DefaultBrandingProvider`. ([e8ebdd2718](https://github.com/elsa-workflows/elsa-studio/commit/e8ebdd2718)) ([#698](https://github.com/elsa-workflows/elsa-studio/pull/698))

- **Login page now uses Radzen components**: The login page was redesigned using Radzen.Blazor (replacing the previous MudBlazor-based layout). Applications that customised the login layout directly may need to revisit their overrides. `MudDialogProvider` has been moved to `BasicLayout`. ([dc1531db32](https://github.com/elsa-workflows/elsa-studio/commit/dc1531db32)) ([#696](https://github.com/elsa-workflows/elsa-studio/pull/696))

## ✨ New Features

#### Workflow Editor

- **Workflow Code View**: A JSON/YAML source-code view is now available directly inside the Workflow Editor. Users can edit the raw workflow definition and optionally have changes auto-saved or auto-applied. Adds `WorkflowDefinitionOptions` with `AutoSaveChangesByDefault` and `AutoApplyCodeViewChangesByDefault` properties for host-level configuration. ([23f828f05d](https://github.com/elsa-workflows/elsa-studio/commit/23f828f05d)) ([#711](https://github.com/elsa-workflows/elsa-studio/pull/711))

- **Monaco Dialog Code Editor**: A modal full-screen Monaco editor is now available from `ExpressionInput` and `Code` input components via a dedicated edit button, enabling comfortable editing of scripts and expressions in a larger surface. ([fa8781a571](https://github.com/elsa-workflows/elsa-studio/commit/fa8781a571)) ([#702](https://github.com/elsa-workflows/elsa-studio/pull/702))

- **Save As**: A "Save As" action is available in the Workflow Editor toolbar, letting users clone a workflow into a new definition without leaving the editor. The previous "Duplicate" action has been refactored as "Clone" behind a shared service. ([4a11287c14](https://github.com/elsa-workflows/elsa-studio/commit/4a11287c14)) ([#625](https://github.com/elsa-workflows/elsa-studio/pull/625))

- **Keyboard Shortcut to Save**: Workflows can now be saved with **Ctrl+S** (Windows/Linux) or **Cmd+S** (macOS) directly from the Workflow Editor. ([78a9f11eb3](https://github.com/elsa-workflows/elsa-studio/commit/78a9f11eb3)) ([#635](https://github.com/elsa-workflows/elsa-studio/pull/635))

- **Export Dialog with Referencing Workflows**: When exporting a workflow definition, users can now opt to include all referencing workflows in the export bundle via a new option in the export dialog. ([997a2dcb95](https://github.com/elsa-workflows/elsa-studio/commit/997a2dcb95)) ([#762](https://github.com/elsa-workflows/elsa-studio/pull/762))

#### Workflow Definition & Instance Lists

- **URL Query Parameter State + Instance Polling**: `WorkflowDefinitionList` and `WorkflowInstanceList` now synchronize their pagination and filter state with the browser URL query string, enabling deep linking, bookmarking, and back-navigation. Adds a user-controllable polling toggle, configurable polling interval, a Refresh menu item, and **Ctrl/Cmd+R** support for manual refresh. ([22d6b5dff4](https://github.com/elsa-workflows/elsa-studio/commit/22d6b5dff4)) ([#700](https://github.com/elsa-workflows/elsa-studio/pull/700))

#### Activity & Workflow Descriptions

- **Markdown Descriptions**: Workflow and activity descriptions now render as Markdown. Introduces reusable `MarkdownEditor` and `MarkdownTextArea` components backed by MudBlazor/Radzen, with support for previewing, read-only mode, and localization. Long descriptions in the definition list are truncated with a "view more" option. ([76ca1d8df1](https://github.com/elsa-workflows/elsa-studio/commit/76ca1d8df1)) ([#659](https://github.com/elsa-workflows/elsa-studio/pull/659))

#### Branding & Customisation

- **Configurable Branding**: `IBrandingProvider` has been significantly extended to allow white-labelling of Elsa Studio — configure the app name, version, tagline, login page background images (with dark mode variants), and app bar icon visibility. `DefaultBrandingProvider` supplies sensible defaults and supports file-based logo/background overrides. ([e8ebdd2718](https://github.com/elsa-workflows/elsa-studio/commit/e8ebdd2718)) ([#698](https://github.com/elsa-workflows/elsa-studio/pull/698))

#### Platform

- **[.NET 10 LTS support](https://github.com/elsa-workflows/elsa-studio/pull/674)**: All Elsa Studio NuGet packages now multi-target `net8.0`, `net9.0`, and `net10.0`. Packages include assemblies for all three frameworks. CI/CD updated accordingly. ([3e4ba147b5](https://github.com/elsa-workflows/elsa-studio/commit/3e4ba147b5)) ([#674](https://github.com/elsa-workflows/elsa-studio/pull/674))

## 🔧 Improvements

#### Performance

- **Workflow Designer: Batched Activity Size Calculation**: Addresses a severe performance regression (3–5× slower rendering for large workflows) by batching DOM operations and sharing a single `requestAnimationFrame` loop across all activities instead of one per activity. Adds accurate cache keying by activity type, display characteristics, and port count. Workflows with duplicate activity types benefit from immediate cache hits (10–100× faster). Saved workflows skip redundant calculations when valid sizes are already stored in metadata. ([d66dad0b47](https://github.com/elsa-workflows/elsa-studio/commit/d66dad0b47)) ([#686](https://github.com/elsa-workflows/elsa-studio/pull/686))

#### Workflow Designer

- **Minimum Node Size Enforcement**: Activity nodes now enforce a minimum size based on content (display text and description length, port count). Size changes are cached and updates are skipped when the new size is unchanged, reducing unnecessary reflows. ([0b02c67257](https://github.com/elsa-workflows/elsa-studio/commit/0b02c67257)) ([#733](https://github.com/elsa-workflows/elsa-studio/pull/733))

- **Prevent Autosave on Programmatic Node Resizing**: Initial-render resizing of activity nodes no longer triggers an autosave. A per-graph suppression flag (with try/finally cleanup) gates graph-update events during programmatic adjustments. Size changes smaller than 0.5 px are also ignored to reduce redundant events. ([663812ca8e](https://github.com/elsa-workflows/elsa-studio/commit/663812ca8e)) ([#766](https://github.com/elsa-workflows/elsa-studio/pull/766))

- **Unified Workflow Export Service**: Introduces `IWorkflowExportDialogService` and `WorkflowExportDialogService` to centralise export-dialog logic across all workflow views (definition list, editor, version viewer). Exported files now consistently use the backend-provided filename. Bulk-export preserves the current selection when the dialog is cancelled. ([aba46df2d4](https://github.com/elsa-workflows/elsa-studio/commit/aba46df2d4)) ([#764](https://github.com/elsa-workflows/elsa-studio/pull/764))

#### Workflow Instance Viewer

- **Workflow State JSON in Details Tab**: The raw workflow state is now exposed as formatted JSON in the Instance Details tab, making it easier to inspect custom state properties. ([fb10077fd8](https://github.com/elsa-workflows/elsa-studio/commit/fb10077fd8)) ([#603](https://github.com/elsa-workflows/elsa-studio/pull/603))

- **Variable ID on Hover**: The variable ID is now shown as a tooltip when hovering the label/value in the `WorkflowInstanceViewer`. ([6a38442381](https://github.com/elsa-workflows/elsa-studio/commit/6a38442381))

#### Activity Properties Panel

- **Fault Exception Messages in Test Tab**: The Test tab now surfaces fault exception messages when a workflow test run faults, reducing the need to dig through server logs. ([d8f6a4c6fc](https://github.com/elsa-workflows/elsa-studio/commit/d8f6a4c6fc)) ([#612](https://github.com/elsa-workflows/elsa-studio/pull/612))

- **Activity Version in Info Tab**: The activity version is now visible in the activity Info tab. ([d0f90c99a8](https://github.com/elsa-workflows/elsa-studio/commit/d0f90c99a8))

#### Resilience / Feature Detection

- **Graceful Resilience Feature Handling**: `RemoteResilienceStrategyCatalog` and `RemoteActivityExecutionService` now check `IRemoteFeatureProvider` before calling Resilience API endpoints. When the Resilience feature is not enabled on the Elsa Server, API calls are skipped and the UI hides the related tab — no more 404 errors surfacing to users. Results are cached for efficiency. ([fd8ce9b3db](https://github.com/elsa-workflows/elsa-studio/commit/fd8ce9b3db)) ([#753](https://github.com/elsa-workflows/elsa-studio/pull/753)) ([856481155](https://github.com/elsa-workflows/elsa-studio/commit/8564811559))

#### UI

- **Modernised Login Page**: The login page has been redesigned with a two-column card layout, background image, and Radzen.Blazor components. Google Fonts imports have been removed from all HTML host files. ([dc1531db32](https://github.com/elsa-workflows/elsa-studio/commit/dc1531db32)) ([#696](https://github.com/elsa-workflows/elsa-studio/pull/696))

- **Compact UI for Tables and Menus**: Applied Dense and Small icon sizing to data panels, menus, and table toolbars across `WorkflowDefinitionList`, `WorkflowInstanceList`, and the Workflow Editor for a cleaner, tighter layout. ([7f557eecfc](https://github.com/elsa-workflows/elsa-studio/commit/7f557eecfc)) ([8b2228db7e](https://github.com/elsa-workflows/elsa-studio/commit/8b2228db7e))

- **Aligned InfoTab and ActivityDetailsTab**: The Info tab in `WorkflowInstanceViewer` and the Activity Details tab now share a consistent field layout and UX conventions. ([b0dae44eb6](https://github.com/elsa-workflows/elsa-studio/commit/b0dae44eb6))

#### Reliability

- **Thread Safety in WorkflowEditor and Workspace**: `StateHasChanged()` calls have been replaced with `await InvokeAsync(StateHasChanged)` across `WorkflowEditor` and `WorkflowDefinitionWorkspace` to prevent threading exceptions in Blazor Server. ([ad059f93ce](https://github.com/elsa-workflows/elsa-studio/commit/ad059f93ce))

## 🐛 Fixes

- **Workflow-as-Activity Instances Not Loading**: Clicking a workflow-as-activity instance in the journal, or double-clicking it on the canvas, now correctly navigates to the sub-workflow instance. Root cause: `IsReadOnly` was incorrectly set to `false` during initialization, suppressing the `ActivityDoubleClicked` event. Journal navigation to activities in nested containers is also restored. ([2fd57289f4](https://github.com/elsa-workflows/elsa-studio/commit/2fd57289f4)) ([#727](https://github.com/elsa-workflows/elsa-studio/pull/727))

- **Extra Flowchart Node on Canvas**: Resolved a regression where an extra `Flowchart` activity node appeared on the design surface when adding container activities. The fix explicitly checks the activity type string `"Elsa.Flowchart"` to avoid treating embedded flowcharts as top-level containers. ([8f1f04744f](https://github.com/elsa-workflows/elsa-studio/commit/8f1f04744f)) ([#669](https://github.com/elsa-workflows/elsa-studio/pull/669))

- **InputsTab "One-Interaction-Late" UI Updates**: Typing in or switching input types in the activity Properties panel no longer required an extra interaction before the UI reflected the change. `InputDisplayModels` are now rebuilt only when the activity or descriptor actually changes, not on every parameter value change. ([5bc18fdc08](https://github.com/elsa-workflows/elsa-studio/commit/5bc18fdc08)) ([#670](https://github.com/elsa-workflows/elsa-studio/pull/670))

- **Monaco Activity Selection Race Condition (WASM)**: Rapidly switching activities in the properties panel no longer throws a `JSException: Couldn't find the editor with id` in Blazor WASM. Fixed by adding `@key="@SelectedActivityId"` to force full component recreation and sequencing async render passes to ensure disposal completes before re-creation. ([d254dad1bc](https://github.com/elsa-workflows/elsa-studio/commit/d254dad1bc)) ([#756](https://github.com/elsa-workflows/elsa-studio/pull/756))

- **Pagination State Lost on Re-render**: The current page is now preserved when `WorkflowDefinitionList` re-renders, preventing users from being sent back to page 1 unexpectedly. Query parameter handling in `QueryTableComponentBase` was also refactored for maintainability. ([11d628f23a](https://github.com/elsa-workflows/elsa-studio/commit/11d628f23a)) ([af8442871c](https://github.com/elsa-workflows/elsa-studio/commit/af8442871c)) ([#735](https://github.com/elsa-workflows/elsa-studio/pull/735)) ([#736](https://github.com/elsa-workflows/elsa-studio/pull/736))

- **Read-Only Tooltip Clipped in Monaco Editor**: The read-only indicator tooltip in the Monaco editor no longer clips outside the component boundaries. ([191691198c](https://github.com/elsa-workflows/elsa-studio/commit/191691198c)) ([#763](https://github.com/elsa-workflows/elsa-studio/pull/763))

- **CodeEditorDialog IntelliSense Popup Offset**: The IntelliSense/autocomplete popup in `CodeEditorDialog` now appears at the correct position. ([c119340659](https://github.com/elsa-workflows/elsa-studio/commit/c119340659))

- **Workflow Export Filenames**: Exported workflow files now consistently use the filename provided by the backend instead of a locally generated kebab-case name. ([aba46df2d4](https://github.com/elsa-workflows/elsa-studio/commit/aba46df2d4)) ([#764](https://github.com/elsa-workflows/elsa-studio/pull/764))
